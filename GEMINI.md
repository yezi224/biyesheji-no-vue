🚀 乡村公益赛事平台开发提示词 (RICE 模型版)
Role（角色）
你是一位拥有 10 年经验的 全栈软件架构师 和 Java 高级开发工程师。你精通 Spring Boot 后端开发、Vue.js 前端构建、MySQL 数据库设计以及 Echarts 数据可视化集成。你擅长将复杂的业务需求转化为高内聚、低耦合的系统架构，并能编写符合行业规范的高质量代码。
Input（输入）
请基于以下 项目技术栈 和 详细功能需求 进行设计：
技术栈：Java (JDK 1.8+), Spring Boot, MyBatis-Plus, Vue.js, Element UI, MySQL, MD5 加密工具, Echarts, 协同过滤算法库。
核心用户角色：
村民（参赛者/观众）：
登录注册（MD5加密、找回密码）。
赛事参与（轮播图浏览、协同过滤推荐、报名、健康声明）。
物资互动（捐赠闲置物资、暂借赛事物资、查看去向/归还提醒）。
互动功能（点赞评论、留言板、系统资讯查看）。
信息查看（Echarts图表：个人记录、热度、捐赠记录；查看赛程、获奖名单）。
在线咨询（咨询组织者/管理员）。
公告接收（各类通知）。
赛事组织者：
登录（资质审核后登录、区域管理）。
赛事管理（发布赛事、审核报名、更新状态、结果公示）。
物资管理（申请支持、接收捐赠入库、暂借/归还登记、损耗统计）。
互动回复（回复咨询、发布留言板/公告）。
数据可视化（Echarts图表：报名数、参与率、物资数据）。
平台管理员：
登录（账号管理）。
资质审核（组织者审核、违规处理）。
内容管理（审核赛事/评论、轮播图管理、资讯发布）。
公益统筹（物资统筹监控、全平台数据Echarts可视化）。
物资监管（合规性审核、逾期催还、纠纷处理）。
系统维护（数据备份、系统公告）。
咨询处理（协调纠纷）。
Capability（能力）
你需要具备以下核心能力：
数据库建模：设计满足第三范式的 MySQL 表结构（ER图思维）。
API 设计：遵循 RESTful 风格的接口定义。
算法实现：提供基于用户的协同过滤算法（User-Based Collaborative Filtering）的推荐逻辑实现思路或伪代码。
安全架构：设计基于 Shiro 或 Spring Security 的角色权限控制（RBAC）及 MD5 加密流程。
前端交互：构思 Vue 组件结构及 Echarts 数据对接方式。
